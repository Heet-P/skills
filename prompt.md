Build a very clean, minimal, modern website for an open-source collection of AI engineering “skills” / “system prompts” designed for production-grade software development workflows.

The website should present these skills as powerful drop-in engineering copilots that developers can use inside tools like:

* Claude Code
* Codex
* Cursor
* Roo Code
* Cline
* OpenHands
* AI coding agents
* internal engineering assistants

The core idea of the platform is:

“Production-grade AI engineering skills that help developers build safer, more scalable, more secure, and more reliable applications.”

The site should showcase several categories of skills such as:

* Security Audit Skills
* OWASP & Adversarial Testing Skills
* Scalability & Architecture Skills
* Reliability & Failure Analysis Skills
* Legal & Compliance Skills
* Dependency & Supply Chain Audit Skills
* DevOps & Production Readiness Skills
* Payment & Fraud Analysis Skills

The website should explain that these are NOT generic prompts.
They are deeply engineered production-focused AI skills designed to:

* scan entire codebases
* understand architecture
* identify vulnerabilities
* test edge cases
* validate security
* analyze scalability
* enforce zero-trust principles
* detect broken workflows
* review compliance risks
* improve production readiness

The messaging should feel:

* engineering-focused
* trustworthy
* developer-first
* practical
* technical but approachable

The website should communicate ideas like:

* “Think like a real security engineer”
* “Test sad paths, not just happy paths”
* “Never trust the browser as a security boundary”
* “Production-grade AI workflows”
* “Security, scalability, and reliability built into development”
* “Drop these skills into your AI coding workflow”

The primary CTA of the website should redirect users to a GitHub repository where they can:

* copy the skills
* use them in their own projects
* contribute new skills
* fork the repository
* integrate them into AI coding agents

The GitHub repo should feel like the central product.

The website should also explain:

* why these skills exist
* what problems they solve
* why normal AI coding workflows miss critical production issues
* how these skills improve real-world engineering quality

Potential sections/pages:

* Hero section
* What These Skills Do
* Skill Categories
* Example Engineering Problems They Catch
* Why Traditional AI Coding Misses These Issues
* Real Production Failure Scenarios
* Security & OWASP Focus
* Architecture & Scalability Focus
* Legal & Compliance Focus
* GitHub CTA
* Open Source / Community Contribution section

The website should feel like:

* an open-source engineering toolkit
* a professional developer utility
* a production engineering resource hub
* an AI engineering quality framework

DO NOT make it feel:

* overly corporate
* flashy
* startup-marketing-heavy
* cluttered
* gimmicky

The focus is:
clarity, trust, engineering quality, production readiness, and practical developer value.

The website should clearly communicate that:
these skills are designed to help AI-generated code become safer and more production-ready before deployment.
